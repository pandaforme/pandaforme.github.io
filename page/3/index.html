[LeetCode] ZigZag Conversion

Posted on 2016-12-09 | Edited on 2016-12-13 | In LeetCode |

題目：

ZigZag Conversion


The string “PAYPALISHIRING” is written in a zigzag pattern on a given number of rows like this: (you may want to display this pattern in a fixed font for better legibility)

P A H N
A P L S I I G
Y I R
And then read line by line: “PAHNAPLSIIGYIR”
Write the code that will take a string and make this conversion given a number of rows:

string convert(string text, int nRows);
convert(“PAYPALISHIRING”, 3) should return “PAHNAPLSIIGYIR”.

Return 3.

解法：

ZigZag是類似N或Z型的走法，根據此題要字串走類似N型走法，然後在一列列的印出。

先從幾個範例中看能不能找到規則：

  • $n = 4$

  • $n = 5$

以$n = 5$為例，發現到

  1. P(標示紅色)到I(標示紅色)中間隔了$2n-3$個字元。
  2. P(標示紅色)的下一個字元A到I(標示紅色)的上一個字元s，就變成$n = 4$的子問題。

可以透過以上的規則寫成程式，當然這樣的規則不是絕對，可以看看其他人的解法，會發現許多不同的面向。

[Java] lock、tryLock和lockInterruptibly的差別

Posted on 2016-12-09 | Edited on 2016-12-13 | In Java |

在Java中的Lock介面有lock、tryLock和lockInterruptibly三個功能很相近的method，看完javadoc後還是搞不太清楚它們之間的差異性，終於找到一篇解釋清楚的文章了。

  • lock()：若lock被thread A取得，thread B會進入block狀態，直到取得lock。
  • tryLock()：若當下不能取得lock，thread就會放棄。
  • lockInterruptibly()：跟lock()情況一下，但是thread B可以透過interrupt被喚醒處理InterruptedException。

範例：

  • lock()：

可以發現ThreadB呈現block狀態，一直在等待Thread A釋放lock。

  • tryLock()：

若一開始lock被Thread A取得，Thread B透過tryLock()當下若沒有取得到lock，就會放棄。

  • lockInterruptibly()：

Thread A先取得lock，Thread B無法取得lock進入block狀態，可以透過發出interrupt方式喚醒Thread B。

Lock和synchronized幾乎是大同小異，但是Lock可以做更細微的同步方式。

    [LeetCode] Palindrome Number

    Posted on 2016-12-09 | Edited on 2016-12-13 | In LeetCode |

    題目：

    Palindrome Number


    Determine whether an integer is a palindrome. Do this without extra space.

    解法：

    題目要求不能有多餘的變數，推測應該是不准將數字轉成字串。

    • 一開始的做法是先比較第一個和最後一個的數字是否相同，若是相同在比較第二個和倒數第二個數字，直到全部比完或是只剩一個數字。

      例如：數字12321

      • $12321 / 10000 = 1$ 可以獲得第一個數字
      • $12321 \bmod 10 = 1$ 可以獲得最後一個數字

      再遞迴執行數字232。

      但是這個做法有漏洞，假如是10011會認為是true，但是實際卻是false。

    • 後來將做法改成：將數字倒轉，再和原本數字比較是否相同。

    Find running median from a stream of integers

    Posted on 2016-12-09 | In Algorithm |

    依序輸入多個數字，如何有效率找出中位數(median)？

    例如：依序輸入7、13、2、6、9，中位數為7。

    解法：

    1. 將所有數字排序，即可以得出中位數。
      時間複雜度為$O(nlogn)$

    2. 利用max heap和min heap：

      • 新增數字到heap：
        • 若max heap為空，數字新增到min heap。
        • 若max heap不為空並且數字小於max heap的root，數字新增到max heap。
        • 若max heap不為空並且數字大於max heap的root，數字新增到min heap。
      • rebalance：
        • 若max heap size - min heap size > 1, 刪除max heap的root並且新增到min heap。
        • 若min heap size - max heap size > 1, 刪除min heap的root並且新增到max heap。
      • 計算中位數：
        • 若max heap size > min heap size，中位數為max heap的root。
        • 若max heap size < min heap size，中位數為min heap的root。
        • 若max heap size == min heap size，中位數為$(max\ heap的root + min\ heap的root) / 2$ (根據定義決定)。

      時間複雜度為$O(1)$

      此方法是利用中位數的特性：

      • 左半部的數字一定小部右半邊。

        例如：2、3、7、13、15 或是2、3、7、13、15

      • 左半部和右半部的大小差值不能超過1，否則不能透過上述方法找出中位數。

      • 左半部的數字(包含中位數)建成max heap，max heap的root即為中位數。

        例如：2、3、7、13、15

        2、3、7建成max heap，13、15建成min heap

      • 右半部的數字(包含中位數)建成min heap，min heap的root即為中位數。

        例如：2、3、7、13、15

        2、3建成max heap，7、13、15建成min heap

    程式碼：

    FindMedian

    [LeetCode] Longest Common Prefix

    Posted on 2016-12-09 | Edited on 2016-12-13 | In LeetCode |

    題目：

    Longest Common Prefix


    Write a function to find the longest common prefix string amongst an array of strings.

    ####解法：
    題目相當簡短，剛開始看不是那麼好懂。

    舉例來說：

    • {“abcde”, “abc”, “ab”}，則Longest Common Prefix為”ab”。
    • {“123456”, “123”, “456”}，則Longest Common Prefix為””。

    可以推論出Longest Common Prefix的最大長度為所有字串中最小長度，拿最小長度字串的每個字元跟其他字串一一比較，可求出Longest Common Prefix。

    假設有m個字串，字串最大長度為n。對字串做排序的時間複雜度為$O(mlogm)$，拿最小長度字串的每個字元跟其他字串一一比較的時間複雜度為$O(mn)$，全部的時間複雜度為$O(mlogm) + O(mn)$。

    若m遠大於n，則全部的時間複雜度為$O(mlogm)$。
    若m遠小於n，則全部的時間複雜度為$O(mn)$。

    程式碼：Longest Common Prefix
